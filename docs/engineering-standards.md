# Aurora FC Engineering Standards

## Code

- Follow PEP8
- Type hints are encouraged
- Functions should be small
- Avoid duplicate code

## Git

- Use Conventional Commits
- Never commit directly to main for major features
- Every feature gets its own branch

## Documentation

- Every module has a README
- Every important decision is documented

## Testing

- Every important module should have tests

## Logging

- Never use print() in production code
- Always use logging
